# 블로그 페르소나 — (브랜드명 미정)

> **태그라인 (가안):** "데이터로 읽는 부동산"
> **URL (가안):** 네이버 블로그 (도메인 없음, 무료 운영)

---

## 1. 블로그 성격

**데이터 기반 부동산 분석 블로그.**
공공 데이터(국토부 실거래가·KB시세·KOSIS 등)를 직접 수집·분석해 차트와 표로 보여준다. 호가나 추측이 아니라 **실거래 데이터로 검증된 사실**만 다룬다. AI 이미지 없음 — 모든 시각 자료는 자동 생성된 차트.

**핵심 가치:** 신뢰, 데이터, 의사결정 보조

기존 주식 블로그(playandplay)와 달리 부동산은 **장기 의사결정**을 다루기 때문에 톤은 더 차분하고 분석적. 단, 어렵지 않게.

## 2. 타겟 독자 (듀얼 페르소나)

### A. 실수요자
| 항목 | 내용 |
|------|------|
| 연령 | 30~45세 |
| 상황 | 내 집 마련, 갈아타기, 신혼/생애최초 |
| 관심 | 학군·교통·생활권·안전 자산 |
| 니즈 | "이 단지 살까 말까, 데이터로 확인하고 싶다" |
| 검색 | 단지명, 동네 이름 |

### B. 투자자
| 항목 | 내용 |
|------|------|
| 연령 | 30~50세 |
| 상황 | 갭투자, 수익형, 재건축 진입 |
| 관심 | 수익률·전세가율·매크로·정책·법인 매수 |
| 니즈 | "이 지역 들어갈까, 시장 신호가 어떤가" |
| 검색 | 지역+키워드 (수익률, 갭, GTX, 재개발) |

70% 데이터 공통, 30%만 관점이 다름. 핵심 콘텐츠는 한 글에 양쪽 섹션 분리.

## 3. 말투 & 페르소나

**컨셉:** 데이터로 차분히 짚어주는 분석가. 친근하지만 가볍지 않게.

| 요소 | 적용 |
|------|------|
| 1인칭 | 자제 — "분석해보면", "데이터를 보면" 같은 데이터 주어 |
| 문장 | 서술형 + 표·차트가 보조. 모바일 가독성 위해 짧은 문단 |
| 톤 | 차분, 분석적. 단, 전문용어 남발 금지 |
| 줄바꿈 | 자주. 모바일 가독성 우선 |
| 이모지 | 최소화. 핵심 신호(📈 🚨 ⚠️)에만 절제 사용 |
| 강조 | **볼드**는 단지명·핵심 수치·결론에만 |
| 금지 | "대박", "급등각", "반드시 오른다" 같은 단정·선동 |

**비교:**
```
❌ (선동·단정)
이 단지는 무조건 오른다. 지금이 들어갈 타이밍.

✅ (분석적·중립)
최근 6개월 평균 거래가는 ㎡당 X만원으로 자치구 평균 대비 +12%.
다만 거래량은 평월 대비 -30%로 매도 우위 신호.
```

**신뢰도를 높이는 표현:**
- "데이터를 보면..."
- "최근 X개월 거래 N건을 집계하면..."
- "다만 한 가지 주의할 점은..."
- "단정하긴 어렵지만, 신호는..."
- "실수요 관점에선 X, 투자 관점에선 Y"

## 4. 콘텐츠 카테고리 (요일별)

`_design.md`의 7시리즈를 그대로 따른다.

- 월: 주간 실거래가 트래커
- 화: 단지 톺아보기 (서울)
- 수: 동네 생활권 분석
- 목: 단지 톺아보기 (수도권)
- 금: 청약·호재 추적
- 토: 데이터 인사이트
- 일: 비교 분석

발행 비율은 단계별로 늘어남 (1단계 주3회 → 3단계 일일).

## 5. 포스트 구조 — 단지 톺아보기 템플릿 (대표 시리즈)

```
[썸네일: 단지 외관 PNG 또는 시세 차트 미니]

# {단지명} — 데이터로 본 시세·입지·전망

도입 (1~2문장): 이 단지를 다루는 이유 / 최근 시세 흐름 한 줄

## 단지 개요
- 위치: {법정동}, 세대수, 건축년도, 동/세대 구성
- 위치 표 + 한 줄 설명

## 시세 추이
- {차트: 최근 12개월 거래 시계열}
- 평균가, 최고가, 최저가 (만원/평 표시)

## 평형별 시세 분포
- {차트: 평형별 박스플롯 또는 막대}

## 인근 비교
- 반경 1km 유사 평형 단지 시세 표

## 실수요 관점
- 학군·교통·생활권·관리 상태
- 입주·갈아타기 시 체크포인트

## 투자 관점
- 전세가율, 갭, 수익률
- 재건축·리모델링 진척

## 한줄평
- 실수요: ...
- 투자: ...

[데이터 출처: 국토부 실거래가 (YYYY-MM 기준)]
```

각 시리즈마다 비슷한 구조 템플릿을 둠 → LLM 자동 생성 일관성 확보.

## 6. 차별화 포인트

| 기존 부동산 블로그 | 우리 |
|------------------|------|
| 호가·추측·언론 인용 | **실거래 데이터** 기반 |
| AI 이미지 / 인터넷 사진 | 직접 생성 차트 |
| 한 관점 (실수요 또는 투자) | **양쪽 관점 한 글에 분리** |
| 단발성 글 | 요일별 시리즈 → 반복 방문 |
| "오를까 떨어질까" 단정 | 데이터 신호 + 한계 명시 |

## 8. AI 활용 워크플로우 (playandplay 재활용)

```
Hermes (CEO) — 기획·검수
  └─ Claude Code — 데이터 수집·분석·작성·차트·발행
```

**프로세스:**
1. Hermes가 기획 (이번 주 단지·동네·시리즈 결정)
2. Claude Code가 데이터 수집 (API 호출, CSV 적재)
3. Claude Code가 분석·차트 생성·.md 작성
4. Hermes 검수 (수치·톤·링크)
5. Claude Code가 .html 변환 + git commit
6. Claude Code 또는 사람이 네이버 발행

## 9. 브랜드명 후보 (확정 필요)

써봄(써보고+ㅁ) 처럼 **동사+ㅁ 명사화 패턴**이 일관성 좋음. 부동산은 데이터 시각화·분석이 정체성이라 시각/분석 동사 후보:

| 후보 | 의미 | 어울림 |
|------|------|--------|
| **그려봄** | 그려보고 보여준다 | ⭐ 데이터 시각화 정체성에 직결 |
| **짚어봄** | 핵심을 짚는다 | ⭐ 분석가 톤 |
| **헤아려봄** | 깊이 헤아린다 | 분석적 but 약간 무거움 |
| **찾아봄** | 정보를 찾아준다 | 무난, 차별성 약함 |
| **재어봄** | 데이터로 재본다 | 측정·정량 느낌 |
| **봐봄** | 직접 본다 | 너무 짧고 단조 |

**1순위 추천: `그려봄`**
- 차트로 부동산을 그려서 보여준다는 정체성과 직결
- 부드러운 어감, 친근함
- "그려봄 부동산" 또는 "그려봄"이 단독으로 브랜드 가능

**2순위: `짚어봄`**
- 분석가 포지션이 명확
- "오를까 짚어봄", "강남 짚어봄" 식의 자연스러운 활용

**확정 후 본 문서 제목과 _design.md 모두 업데이트.**
