# 부동산 블로그 — 디자인 문서

> 데이터 기반 부동산 분석 블로그. 실수요자(A) + 투자자(B) 듀얼 페르소나, Git 기반 자동화 파이프라인, 일일 발행 도달이 목표.

작성일: 2026-05-05
상태: PoC 검증 완료 → 본 디자인 문서 작성

---

## 1. 컨셉 한 줄

**"실거래 데이터로 읽는 부동산 — 차트와 비교로 실수요·투자 의사결정을 돕는다."**

기존 블로그 차별점:
- AI 이미지 없음. 모든 시각 자료는 **공공 데이터에서 자동 생성된 차트·표**
- 단지·동네·자치구 단위로 **롱테일 SEO 자산** 누적
- 실수요·투자 두 관점을 **한 글 안에서 분리 섹션**으로 처리

## 2. 타겟 독자 (A + B)

| 구분 | A: 실수요자 | B: 투자자 |
|------|-----------|---------|
| 연령 | 30~45세 | 30~50세 |
| 상황 | 내 집 마련, 갈아타기 | 갭투자, 수익형, 재건축 |
| 관심 | 학군·교통·생활권·안전 자산 | 수익률·전세가율·매크로·정책 |
| 읽는 이유 | "이 단지 살까 말까" | "이 지역 들어갈까 말까" |
| 검색 패턴 | 단지명·동네 이름 | 지역+키워드 (수익률, 갭, GTX) |

70%의 데이터가 양쪽에 공통, 30%만 관점이 다름. 핵심 콘텐츠는 한 글에 **실수요 섹션 + 투자 섹션**을 분리 배치.

## 3. 콘텐츠 전략 — 요일별 7시리즈

각 시리즈는 주 1회만 발행되어 일일 발행을 분산. 단계적 출범으로 자동화 파이프라인을 안정시킨 후 풀 가동.

| 요일 | 시리즈 | 페르소나 | 차트 유형 | 콘텐츠 풀 | 출범 단계 |
|------|-------|---------|---------|---------|---------|
| 월 | 주간 실거래가 트래커 | A+B | 자치구 히트맵, TOP10 | ∞ (매주 갱신) | 1단계 |
| 화 | 단지 톺아보기 (서울) | A+B 듀얼 | 시세 추이, 평형별, 인근 비교 | 서울 ~3,000 단지 | 1단계 |
| 수 | 동네 생활권 분석 | A 중심 | 인구 변동, 가격 분포 | 서울 467동 + 수도권 ~1,500 | 2단계 |
| 목 | 단지 톺아보기 (수도권) | A+B 듀얼 | 화요일과 동일 | 수도권 ~5,000 단지 | 1단계 |
| 금 | 청약·호재 추적 | A+B | 경쟁률, 호재 영향 | 시의성 (매주 신규) | 3단계 |
| 토 | 데이터 인사이트 | B 중심 | 매크로 시계열, 상관관계 | 가공·해석 무한 | 3단계 |
| 일 | 비교 분석 (X vs Y) | A+B | 인터랙티브 비교 슬라이더 | 조합 수천만 | 2단계 |

## 4. 시리즈별 상세

### 월요일 — 주간 실거래가 트래커
- 전주(월~일) 실거래 자동 분석 → 신고가/거래량 핫스팟 추출
- 핵심 콘텐츠: "이번주 신고가 TOP 10", 자치구별 거래량·가격 변동률
- 데이터: 국토부 실거래가 API
- 역할: **재방문 엔진** — 매주 같은 시간 발행 → 구독 형성

### 화요일 — 단지 톺아보기 (서울)
- 한 글 = 한 단지 심층 분석
- 글 구조 (확정):
  1. 단지 개요 (입지·교통·세대수·건축년도)
  2. 시세 추이 (최근 1년 거래 시계열 차트)
  3. 평형별 시세 분포
  4. 인근 단지 비교 (반경 1km 내 유사 평형)
  5. **실수요 관점** — 학군·생활권·교통
  6. **투자 관점** — 갭·수익률·재건축 진척
  7. 한줄평
- 데이터: 국토부 실거래가 + KB시세(보조) + 학군 데이터(공개 자료)
- 역할: **롱테일 SEO 핵심** — 각 글이 단지 키워드 점유

### 수요일 — 동네 생활권 분석 (2단계)
- 한 글 = 한 동/읍/면
- 인구 변동, 가격대 분포, 학군, 인프라 종합
- 데이터: KOSIS + 국토부 실거래 + 통계청
- 역할: 실수요자 의사결정 콘텐츠 ("어디서 살까")

### 목요일 — 단지 톺아보기 (수도권)
- 화요일과 동일 형식, 수도권 단지 대상

### 금요일 — 청약·호재 추적 (3단계)
- 이번주 청약 공고 분석 또는 GTX/재개발 호재 영향 추적
- 데이터: 청약홈 + 국토부 고시·보도자료
- 역할: **시의성 트래픽** — 속보형, 검색 의도 명확

### 토요일 — 데이터 인사이트 (3단계)
- 매크로 지표 모니터링: 미분양·거래량·전세가율·금리 + **법인 매수 비율**·**직거래 비율** (PoC에서 확인)
- 데이터: KB월보 + 한국부동산원 R-ONE + 국토부 실거래(파생)
- 역할: 투자자 충성 독자 형성

### 일요일 — 비교 분석 (X vs Y) (2단계)
- "동탄 vs 평택", "마곡 vs 위례" 같은 의사결정형
- 인터랙티브 비교 슬라이더 컴포넌트 활용
- 역할: SNS 바이럴 가능

## 5. 단계별 출범 로드맵

```
1단계 (1~4주차): 파이프라인 검증
  ├─ 화: 단지 톺아보기 (서울)
  ├─ 목: 단지 톺아보기 (수도권)
  └─ 월: 주간 실거래 트래커
  → 주 3회 / 자동화 검증 / 차트 컴포넌트 1차

2단계 (5~12주차): 다양화
  ├─ + 수: 동네 분석
  └─ + 일: 비교 분석
  → 주 5회 / PR 검수 워크플로우 정착

3단계 (3개월+): 일일 도달
  ├─ + 금: 청약·호재
  └─ + 토: 데이터 인사이트
  → 일일 발행 / 데이터 트리거 자동 콘텐츠 실험
```

## 6. 인프라

- **레포**: `github.com/micael17/blog` (기존 모노레포)
- **위치**: `blog/realestate/` (top-level, playandplay와 분리)
- **콘텐츠 포맷**: `.md` (작성용) + `.html` (발행용) + `*.png` (차트·썸네일)
- **글 폴더 컨벤션**: `{topic}-{MMDD}/` (playandplay 패턴 따름)
- **발행 플랫폼**: 네이버 블로그(메인) — 부동산 검색 트래픽 비중상 네이버가 압도적
- **자동 발행**: 미정 (1단계는 수동/반자동, 미래에 브라우저 자동화 검토)
- **메타 DB**: `blog_posts.db` (기존 SQLite 재활용)

## 7. 데이터 소스 (PoC 검증 결과 반영)

| 소스 | 갱신 주기 | 인증 | PoC 검증 | 사용 시리즈 |
|------|---------|------|---------|------------|
| 국토부 실거래가 OpenAPI | 일간 | 공공데이터포털 키 | ✅ 통과 | 월·화·목·일·토(파생) |
| KB시세 / KB월보 | 주/월 | 비공식 (공개 자료 활용) | 미검증 | 화·목·토 |
| 청약홈 | 공고 발생 시 | 공공데이터포털 | 미검증 | 금 |
| KOSIS (통계청) | 월간 | 공공데이터포털 | 미검증 | 수 |
| 한국부동산원 R-ONE | 월간 | 공식 OpenAPI | 미검증 | 토 |
| 국토부 고시 | 비정기 | 웹스크래핑 | 미검증 | 금 |

PoC에서 핵심 소스(국토부 실거래가) 검증 완료. 나머지는 시리즈 출범 직전 단계별 검증.

### 국토부 실거래가에서 확보한 필드 (활용 매핑)
- 단지명·법정동·지번 → 단지 식별
- 전용면적·층·건축년도 → 평형/층/년식 분석
- 거래금액·거래일 → 시세 추이, 신고가
- 매수/매도자 구분 → **법인 매수 비율 시리즈** (3단계)
- 거래유형(직거래/중개) → **직거래 비율 시리즈**
- 해제일·해제구분 → 가격 분석 시 해제 거래 필터링

## 9. 자동화 워크플로우 (playandplay 패턴 재활용)

```
[Hermes 기획]
  └─ "이번 주 단지 톺아보기 대상은 X"
[Claude Code 리서치]
  └─ 국토부 API + KB시세 + 학군 데이터 수집
[Claude Code 분석·작성]
  └─ .md 작성 + 차트 PNG 생성 + .html 변환
[Hermes 검수]
  └─ 사실·톤·링크 확인
[Claude Code 커밋]
  └─ git push (자동 백업)
[수동/반자동 발행]
  └─ 네이버 블로그
```

미래 확장:
- GitHub Actions cron으로 데이터 수집·초안 PR 자동화
- 브라우저 자동화로 네이버 발행 자동화 검토
