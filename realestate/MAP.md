## 2. 요일 × 데이터 사용 매트릭스

`⭐` 주력 데이터 / `✓` 보조 / `─` 사용 안 함

|     | 월<br>주간트래커 | 화·목<br>단지톺아보기 | 수<br>동네분석 | 금<br>청약·호재 | 토<br>인사이트 | 일<br>비교분석 |
|-----|:-:|:-:|:-:|:-:|:-:|:-:|
| **국토부 실거래가** | ⭐ | ⭐ | ⭐ | ─ | ✓ | ⭐ |
| **R-ONE 통계** | ✓ | ✓ | ─ | ─ | ⭐ | ✓ |
| **청약홈** | ─ | ─ | ─ | ⭐ | ─ | ─ |
| **지하철역** | ─ | ✓ | ✓ | ─ | ─ | ✓ |
| **학교알리미** | ─ | ✓ | ⭐ | ─ | ─ | ✓ |
| **KB 주간** | ─ | ✓ | ✓ | ─ | ⭐ | ✓ |
| **국토부 RSS** | ─ | ─ | ─ | ⭐ | ─ | ─ |

**1단계 출범 (월·화·목)** = 국토부 실거래가 1개로 가능 ✅
**2단계 (수·일 추가)** = 지하철역 + 학교알리미 추가 필요
**3단계 (금·토 추가)** = R-ONE + 청약홈 + KB + 국토부 RSS 검증 필요

## 3. 요일별 카드

### 월요일 — 주간 실거래가 트래커
```

주력 데이터:     국토부 실거래가 OpenAPI
글 구조:        자치구별 거래량 → 신고가 TOP10 → 일별 거래량 → 한줄평
출력 차트:      chart-volume-by-region.png
                chart-shingo-top.png
                chart-daily-volume.png
스크립트:       scripts/weekly_tracker.py
독자:           A+B (실수요+투자)
```

### 목요일 — 단지 톺아보기 (수도권)

## 5. 데이터 캐싱 전략

같은 자치구 데이터를 여러 시리즈가 공유 → 1번 호출로 N개 글 생성:

```
data/{lawd_cd}-{ymd}.csv     국토부 실거래 월별 캐시 (gitignore)

월요일: 강남구 4월 → API 1회 호출 → data/11680-202604.csv 저장
화요일: 강남구 단지 톺아보기 → 캐시 재사용 (호출 0회)
일요일: 강남구 vs 서초구 비교 → 강남 캐시 재사용 + 서초만 호출
```

운영 비용 (1단계, 1주):
- API 호출: 자치구별 월 1회 × 7자치구 = **7회/주**
- 일일 한도 1만 회 대비 사실상 무시할 수준

## 6. 비공식 데이터는 쓰지 않는 이유

| 비공식 옵션 | 풍부도 | 안정성 | 합법성 |
|-----------|------|--------|--------|
| KB 내부 API (PublicDataReader) | 높음 | 낮음 (언제든 막힘) | 회색 |
| 네이버 부동산 비공식 | 높음 | 매우 낮음 (rate limit·차단) | 약관 회색 |
| 호갱노노/아실 | 높음 | 낮음 | 회색 |

**원칙**: 자동화 안정성 > 데이터 풍부도. 비공식 막히면 도미노로 글 멈춤.
호가·매물 같은 비공식 정보가 필요하면 글에서 "기사 인용" 식으로 사람이 보강.
