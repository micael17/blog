## 발견한 함정

1. **신청 직후 키 활성화 timing** — 발급 즉시는 401, 수 분 후 200 OK. 첫 시도 실패해도 잠시 대기 후 재시도.
2. **엔드포인트 형식**: 공공데이터포털 페이지의 "End Point" 표기는 service base만 보여줌. 실제 호출엔 operation을 추가해야 함:
   - 페이지 표기: `https://apis.data.go.kr/1613000/RTMSDataSvcAptTrade`
   - 실제 호출: `.../RTMSDataSvcAptTrade/getRTMSDataSvcAptTrade`
   - 패턴: `.../{서비스명}/get{서비스명}`
3. **인증키 전달**: decoding 키를 requests params에 넣으면 자동 URL encoding 됨 → 정상 동작. encoding 키를 직접 URL에 박아도 OK. 이중 encoding만 피하면 됨.
4. **전용면적 단위**: `excluUseAr`는 m² (평 아님). 표시 시 평으로 변환하려면 ÷3.3058.
5. **거래금액 단위**: `dealAmount`는 만원 + 콤마 포함 문자열 (`"320,000"`). 정수 변환 필요.

## 부동산 블로그 디자인에 미치는 영향

### 1단계 시리즈에 직접 활용 가능
- **월요일 - 주간 실거래가 트래커**: 자치구별 호출 → 신고가/거래량 추출 ✅
- **화·목요일 - 단지 톺아보기**: 단지별 거래 이력 추출(여러 달 호출 누적) ✅
- 차트 컴포넌트(시계열, TOP10) 이미 검증됨 ✅

### 추가로 가능한 분석 (디자인에 추가 가치)
- **법인 매수 비율 트래커** (`buyerGbn`) — 투자자 페르소나에게 강력
- **직거래 비율 분석** (`dealingGbn`) — 시장 신호 콘텐츠
- **해제 거래 모니터링** (`cdealType`) — 가격 분석 정확도 ↑

### 데이터 한계
- **현재 시세 ≠ 실거래가**: 호가 정보는 없음. KB시세, 호갱노노 같은 다른 소스 필요할 때 있음.
- **법정동 단위 한계**: 자치구(시군구) 단위 호출만 가능. 더 작은 단위(동) 데이터는 클라이언트에서 필터링.
- **월간 단위**: 한 호출 = 한 자치구 한 달치. 1단지 시계열 분석은 여러 달 호출 + 누적 필요.

## 자동화 파이프라인 신뢰도

**상**: API는 안정적이고 응답 빠름. 무료 한도(1만/일)는 우리 운영에 차고 넘침 (1단계 발행에 50~100콜/일 예상). 정부 공식 데이터라 신뢰성/EEAT 측면 강력.

## 다음 스텝

이 PoC는 **"가능성 검증"** 단계. 실제 1단계 발행으로 가려면:

1. **확장 데이터 수집** — 여러 자치구·여러 달 자동 수집 (캐싱·재시도 로직)
2. **단지 마스터** — 같은 단지명/지번 매칭으로 단지별 시계열 누적
3. **차트 컴포넌트 라이브러리** — 정적 PNG 8~10종 + 인터랙티브 4~5종 설계
4. **글 생성 파이프라인** — Hermes 기획 → Claude Code 글·차트 → md+html
5. **발행 워크플로우** — 네이버 블로그 수동/반자동 (playandplay 패턴 재활용)

## 환경 관리 권장

PoC에서 환경변수로 키를 다뤘는데, 실 운영엔 `.env` 파일이 표준:

```bash
# .env (gitignore 처리)
MOLIT_API_KEY=발급받은_키

# python-dotenv로 로드
from dotenv import load_dotenv
load_dotenv()
```

`.env`는 절대 git에 커밋하지 않고, `.env.example`만 커밋해서 다른 사람이 어떤 키 필요한지 알 수 있게 함.
