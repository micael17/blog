## 맥미니 홈서버 구축기 - 1
### 맥미니에서 웹 서비스 호스팅하기

> 사이드 프로젝트로 만든 웹앱, 밖에서도 접속하게 해보고 싶었다.
> 근데 클라우드 서버 비용이 부담스러워서... 결국 맥미니에 꽂혔다. 💻

---

내가 만든 웹 서비스를 밖에서도 쓰고 싶은데, 매달 클라우드에 돈 내는 게 아깝다고 느낀 적이 있다면 공감할 거다.
나도 그랬다. AWS Lightsail도 써보고, Google Cloud 무료 티어도 써봤는데...

**매달 5만원~10만원** 나가는 게 부담스러웠다.
내가 만든 서비스가 수익이 나는 것도 아니니까. 😅

그러다 생각한 게 **"집에 있는 맥미니로 직접 돌리면 안 되나?"**

지금 그렇게 쓰고 있는데, 3개월째 완전히 만족 중이다.
오늘은 그 경험을 공유해보려고 한다.

### 🔧 기본 구성

내 홈서버 세팅은 이렇게 되어 있다.

```
외부 접속
   ↓
🌐 Cloudflare Tunnel  ← 도메인 연결 + 동적 IP 우회
   ↓
🔄 PM2               ← 서비스 자동 실행 + 재시작
   ↓
💻 맥미니 M2 Pro      ← 본체
```

**실제 사용 스펙:**
- 맥미니 **M2 Pro** (16코어 GPU, 32GB RAM, 512GB SSD) 🖥️
- **macOS Sequoia** (그냥 쓰던 맥 그대로)
- 24시간 켜져 있음 (절전 모드 해제)
- 전기세: **월 3~5천원** 수준

처음엔 "맥을 24시간 켜놔도 괜찮나" 싶었는데,
전기세는 정말 깜짝 놀랄 정도로 적게 나온다.
M칩 맥미니는 아이들 상태에서 **7~10W** 수준이라,
한 달 내내 켜놔도 **일반 데스크탑의 1/10 수준**이다. 🤑

### 첫 번째 기둥: PM2로 서비스 관리하기

홈서버에서 가장 중요한 건 **서비스가 죽었을 때 자동으로 살아나는 것**이다.
내가 24시간 모니터링할 순 없으니까. 😅

그걸 해주는 게 **PM2**다.

PM2는 Node.js 기반의 프로세스 매니저인데,
Node.js 앱뿐만 아니라 **Python 스크립트, 셸 명령어, 어떤 백그라운드 프로세스든** 관리할 수 있다.

**내가 쓰는 PM2 주요 기능:**

| 기능 | 설명 |
|------|------|
| **자동 재시작** | 프로세스가 죽으면 즉시 다시 살림 🔄 |
| **로그 관리** | stdout/stderr 자동 로깅, 로그 로테이션 지원 📋 |
| **시작 스크립트** | 맥 부팅 시 PM2가 자동 실행되게 설정 가능 |
| **모니터링** | CPU/메모리 사용량 실시간 확인 |
| **클러스터 모드** | 멀티코어 활용한 로드밸런싱 |

실제로 PM2로 관리하는 서비스 중 하나 예시를 보면:

```bash
# PM2로 앱 등록하기
pm2 start app.py --name "my-web-app" --interpreter python3

# 현재 상태 보기
pm2 list

# 서비스 모니터링
pm2 monit
```

특히 `pm2 monit` 화면을 보면 CPU, 메모리를 실시간으로 볼 수 있어서
"혹시 서비스가 메모리를 너무 먹고 있나" 체크하기 좋다.

가장 자주 쓰는 명령어:

- `pm2 status` — 서비스 목록과 상태
- `pm2 logs` — 모든 서비스 로그 실시간 확인
- `pm2 restart all` — 업데이트 후 전체 재시작

Node.js 글로벌로 한 번만 설치해두면,
앞으로 추가하는 모든 웹 서비스를 PM2에 등록해서 관리할 수 있다.

**설치와 기본 셋업:**

```bash
npm install -g pm2
```

설치하고 나면 딱 두 가지만 기억하면 된다.

1. **앱 등록:** `pm2 start 파일명 --name "서비스이름"`
2. **시작 스크립트 등록:** `pm2 startup` (맥 부팅 시 자동 실행)

이렇게 해두면, 맥미니를 재부팅해도 PM2가 알아서 모든 서비스를 다시 살려준다.
정말 편하다. 👍

### 두 번째 기둥: Cloudflare Tunnel로 외부 공개하기

서비스는 잘 돌아가는데... 밖에서 접속하려면 어떻게 해야 할까?

클라우드 서버는 고정 IP가 기본 제공되지만,
**일반 가정용 인터넷은 유동 IP**라서 IP가 수시로 바뀐다.
게다가 포트 포워딩도 ISP(인터넷 회사)에 따라 막혀 있는 경우가 많다.

이걸 해결해주는 게 **Cloudflare Tunnel**이다. 🚇

**원리는 간단하다.**

```
사용자 → cloudflare.com/내도메인
              ↓ (Cloudflare 글로벌 네트워크)
         Cloudflare Tunnel
              ↓ (outbound 연결만! 맥미니에서 먼저 연결)
         내 맥미니의 localhost
```

맥미니에서 Cloudflare로 **outbound 연결만** 만들면 끝이다.
방화벽, 포트 포워딩, DDNS 설정이 전혀 필요 없다.
클라우드플레어가 중간에서 알아서 라우팅해준다. ✨

**실제 설정 과정:**

```bash
# cloudflared 설치
brew install cloudflared

# 터널 생성
cloudflared tunnel create my-tunnel

# 터널과 도메인 연결
cloudflared tunnel route dns my-tunnel coloring-playground.com

# config.yml 작성
cloudflared tunnel --config ~/.cloudflared/config.yml run
```

config.yml은 대략 이런 느낌이다.

```yaml
tunnel: my-tunnel
credentials-file: /Users/me/.cloudflared/credentials.json

ingress:
  - hostname: coloring-playground.com
    service: http://localhost:3000
  - hostname: trading.coloring-playground.com
    service: http://localhost:5000
  - service: http_status:404
```

이렇게 해두면 **coloring-playground.com**으로 들어오는 요청이
맥미니의 localhost:3000으로 바로 연결된다.

**Cloudflare Tunnel의 장점 세 가지:**

1. 🛡️ **보안** — 내 집 IP가 완전히 숨겨짐 (DDoS 방어도 Cloudflare가 처리)
2. 🔄 **동적 IP 자동 대응** — IP가 바뀌어도 터널은 문제없음
3. 🔌 **포트 포워딩 제로** — 공유기 설정 건드릴 필요 없음

보안 측면에서 특히 마음에 드는 점은,
외부에서 직접 내 맥미니 IP를 때릴 수가 없다는 거다.
Cloudflare가 중간에서 필터링해주니 **스캐닝 공격 같은 게 아예 들어오지도 않는다.**

### 실제 운영 조합

PM2 + Cloudflare Tunnel을 합쳐서 실제로 운영하는 모습은 이렇다.

```
[PM2로 관리 중인 서비스들]
├── my-web-app       │ port 3000 │ online 3일
├── api-server       │ port 5000 │ online 7일
├── cloudflare-tunnel │ -        │ online 3일  ← 터널도 PM2에 등록!
└── telegram-bot    │ port 8080 │ online 14일
```

눈여겨볼 점은 **cloudflared 자체도 PM2에 등록**해둔 거다.
터널이 혹시 죽으면 PM2가 자동으로 다시 살려준다.
Cloudflare Tunnel + PM2의 조합은 정말 궁합이 좋다. 😄

운영하면서 든 생각은,
"이걸 왜 진작 안 했지?"였다.

### ⚠️ 단점부터 솔직히 말하자면

좋은 점만 말하면 안 되니까, 아쉬운 점도 얘기한다.

**인터넷이 끊기면 서비스도 죽는다**
당연한 얘기지만, 집 인터넷이 불안정하면 서비스도 불안정해진다.
아파트 회선이고 KT라서 크게 끊긴 적은 없지만,
가끔 야간 점검 같은 게 있으면 그 시간에는 접속이 안 된다.

→ 해결책: LTE나 다른 회선을 **백업**으로 두는 방법이 있지만, 아직까진 필요성을 못 느꼈다.

**전원 문제**
집에 있는 전기는 100% 안정적이지 않다.
정전은 거의 없지만, 어댑터가 빠지거나 멀티탭 스위치를 실수로 누르면... 😇

→ 해결책: UPS(무정전 전원장치)를 달면 안심이 된다.
아직 안 샀지만, 언젠가는 사야지 생각 중이다.

**맥미니는 서버 전용 OS가 아니다**
macOS는 서버 OS가 아니다 보니 리눅스 서버에 비해 관리 도구가 제한적이다.
예를 들어 systemd 같은 건 없어서 PM2로 대체해야 한다.
