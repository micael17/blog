## 맥미니로 웹 서비스 운영해보기 (홈서버 셋업 가이드)

평소에 사이드 프로젝트로 만든 웹 서비스를 밖에서도 접속하게 하고 싶은데, AWS나 클라우드 서버 쓰자니 매달 나가는 돈이 아깝다. 나도 그런 고민을 꽤 오래 했었는데, 결론은 **맥미니**였다.

이 글을 다 읽고 나면, 맥미니 한 대로 웹 서비스를 운영하는 전체 그림이 머릿속에 그려질 거다. 💻

---

### ❓ 왜 하필 맥미니인가

클라우드 서버(VPS)랑 집에 두는 맥미니를 비교해보면 이렇다.

| 비교 항목 | 클라우드 서버 (AWS Lightsail 등) | 맥미니 홈서버 |
|-----------|-------------------------------|--------------|
| **월 비용** | 5~50만원 (스펙 따라) | **전기세 3~5천원** |
| **스펙** | vCPU 2~4코어, RAM 1~8GB | **M칩 8~12코어, RAM 16~64GB** |
| **저장소** | SSD 20~80GB | **SSD 256GB~2TB** |
| **GPU** | 없음 (비싼 GPU 인스턴스 제외) | **내장 GPU (머신러닝/영상에도 OK)** |
| **안정성** | 99.9% 보장 | 인터넷/전기 따라감 |
| **고정 IP** | 기본 제공 | 유동 IP (터널 필요) |

단순 비교만 봐도, 같은 돈(거의 공짜)으로 훨씬 강력한 컴퓨터를 쓸 수 있다는 게 맥미니의 장점이다.

<br>

**내가 쓰는 구성은 이렇다.**

> 맥미니 M2 Pro (16코어 GPU, 32GB RAM, 512GB SSD)
>
> 기본형 M1만 돼도 웹 서비스 운영에는 차고 넘친다. M2 Pro는 좀 오버스펙이긴 한데, 나중에 로컬에서 AI 모델도 돌려보려고 넉넉하게 갔다.

### 🛠️ 1단계: 맥미니 세팅

일단 맥미니를 받으면 기본 세팅부터 한다.

**SSD 외장 storage**

512GB 기본 모델이면 금방 찬다. 나는 **삼성 T7 외장 SSD 1TB**를 USB-C로 연결해서 쓰고 있다.

- 맥미니 자체 저장소: 시스템 + 개발 환경
- 외장 SSD: 도커 이미지, 데이터베이스, 로그 파일

> 외장 SSD를 추천하는 이유: 맥미니 기본 SSD는 납땜이라 교체가 안 된다. 저장소는 외장으로 확장하는 게 정신건강에 좋다.

**무결점 셋업 (헤드리스)**

모니터 없이 운용하려면 초기 한 번만 연결해서 세팅하고, 그다음부터는 **화면 공유**(VNC)나 **SSH**로 원격 접속한다.

1. 시스템 설정 → 공유 → 원격 로그인 켜기
2. 시스템 설정 → 공유 → 화면 공유 켜기
3. 맥미니 자동 로그인 설정 (재부팅되어도 알아서 켜지게)
4. 에너지 절약 → '디스플레이가 꺼져 있을 때 자동으로 잠들지 않게' 체크

이렇게 하면 맥미니를 모니터 없이 24시간 돌릴 수 있다.

### 🔧 2단계: 개발 환경 & 프로세스 관리

**Homebrew**로 필요한 거 다 깐다.

```bash
# 필수 패키지
brew install node nginx git
```

**PM2로 프로세스 관리**

웹 서비스를 백그라운드에서 돌리고, 서버가 꺼졌다 켜져도 자동 복구되게 하려면 **PM2**가 진리다.

```bash
npm install -g pm2

# 서비스 등록
pm2 start app.js --name my-service

# 재부팅 시 자동 재시작 설정
pm2 startup
pm2 save
```

내 경우엔 지금 PM2로 3~4개의 서비스를 동시에 돌리고 있다. 증권 데이터 수집 봇도 있고, 내부용 대시보드도 있고. 전부 PM2 하나로 관리하니 편하다.

> [참고] PM2는 Node.js 기반이라 Python/Go 등 다른 언어로 만든 서비스도 `pm2 start` 한 줄로 등록 가능하다.

<br>

### 🌐 3단계: 외부에서 접속하게 만들기

이게 가장 까다로운 부분이다. 일반 가정집 인터넷은 **유동 IP**고, ISP가 포트를 막아놓은 경우도 많다.

**선택지 비교**

| 방법 | 난이도 | 보안 | 속도 | 비용 |
|------|--------|------|------|------|
| 포트포워딩 + DDNS | 중간 | 낮음 | 빠름 | 무료 |
| Cloudflare Tunnel | 쉬움 | **높음** | 중간 | **무료** |
| ngrok | 매우 쉬움 | 중간 | 느림 | 유료 |

**내 선택은 Cloudflare Tunnel이다.**

Cloudflare Tunnel은 방화벽을 열지 않고도 Cloudflare 네트워크를 통해 내부 서버로 터널링해주는 방식이다.

1. Cloudflare 가입 + 도메인 등록
2. cloudflared 설치
3. 터널 생성 → 도메인 연결

```bash
brew install cloudflared

# 터널 생성
cloudflared tunnel create my-tunnel

# 설정 파일
# ~/.cloudflared/config.yml
tunnel: my-tunnel
credentials-file: ~/.cloudflared/my-tunnel.json
ingress:
  - hostname: app.mydomain.com
    service: http://localhost:3000
  - hostname: dashboard.mydomain.com
    service: http://localhost:4000
  - service: http_status:404

# 실행
cloudflared tunnel run my-tunnel
```

이러면 `https://app.mydomain.com`으로 접속하면 내 맥미니의 localhost:3000으로 연결된다. 공인 IP 노출도 없고, SSL 인증서도 Cloudflare가 자동으로 붙여준다. 👍

> 장점: 방화벽을 안 열어도 된다는 게 제일 크다. ISP가 포트 막아놨든 말든 상관없음.
> 단점: Cloudflare 경유이므로 약간의 레이턴시가 생긴다. 하지만 일반 웹 서비스 체감에는 거의 영향 없다.

### 🚀 4단계: 실제 운영 꿀팁

몇 달 돌리면서 깨달은 꿀팁들을 정리해봤다.

**1) PM2 + Cloudflare Tunnel을 system 서비스로 등록**

맥미니가 재부팅되어도 알아서 모든 서비스가 살아나야 진정한 서버다.

```bash
# cloudflared를 brew service로 등록
brew services start cloudflared

# PM2 startup 이미 설정했으면 완료
```

**2) 모니터링은 Telegram 알림으로**

서비스에 문제 생기면 텔레그램으로 알림이 오게 해두면 안심된다.

```python
# 간단한 health check 스크립트 예시
import requests, os
TOKEN = os.getenv('TELEGRAM_BOT_TOKEN')
CHAT_ID = '123456789'

try:
    r = requests.get('http://localhost:3000/health', timeout=10)
    if r.status_code != 200:
        raise Exception(f"Status: {r.status_code}")
except Exception as e:
    url = f"https://api.telegram.org/bot{TOKEN}/sendMessage"
    requests.post(url, json={
        'chat_id': CHAT_ID,
        'text': f'🚨 서비스 다운! {e}'
    })
```

이걸 cron에 5분마다 돌리면 실시간 모니터링 시스템 완성이다.

**3) 자동화 스크립트 활용**

반복 작업은 다 자동화했다. 데이터 수집, 리포트 생성, 백업까지. 맥미니는 24시간 켜져 있으니 cron으로 원하는 시간에 실행시키면 된다.

### ⚠️ 솔직한 단점

장점만 말하면 신뢰가 안 가니까, 실제 써보고 느낀 단점도 정리한다.

| 단점 | 설명 |
|------|------|
| **인터넷 의존성** | 집 인터넷이 끊기면 서비스도 다운된다. (KT가 한 번 먹통 됐을 때 3시간 동안 서비스 장애) |
| **전원 관리** | 정전되면 끝. UPS(무정전 전원장치) 생각해볼 만하다. |
| **발열** | M칩은 발열 적은 편인데도 여름에 50도까지 올라간 적 있다. 환풍 좋은 곳에 두는 게 좋다. |
| **초보자 난이도** | cloudflared, PM2, 터미널 명령어가 익숙하지 않으면 진입장벽이 있다. |
| **고정 IP 필요 서비스** | 간혹 IP 기반 인증하는 외부 API를 써야 할 때 불편하다. |
