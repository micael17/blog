## 1. 블로그 성격

IT 라이프스타일 & 스마트워크 실사용 리뷰 블로그.
제품 스펙 나열이 아니라 **실제로 사서 써보고 느낀 점**을 솔직하게 공유.
광고성 리뷰가 아니라 "이거 살까 말까 고민하는 사람" 기준에서 쓴다.

**핵심 가치:** 신뢰, 실용성, 솔직함

## 3. 말투 & 페르소나 (기존 주식 블로그와 완전히 다른 목소리)

**컨셉:** 옆자리 동기가 추천해주는 느낌

| 요소 | 적용 |
|------|------|
| **1인칭** | 적극 사용 — `내가`, `써보니`, `느낀 점은` |
| **문장** | 구어체 기반 but 너무 가볍지 않게. `~더라`, `~것 같다` 허용 |
| **솔직함** | 단점은 반드시 언급. `이건 좀 아쉬웠다`, `이 가격이면...` |
| **줄바꿈** | 자주. 모바일 가독성 최우선 |
| **이모지** | ✅ 적극 사용 — `💻`, `🖥️`, `🎧`, `⌨️`, `🔋` 등 |
| **강조** | `**볼드**`는 제품명·핵심 스펙·가격에만 |
| **금지어** | 없음. 다만 "대박", "개쩜", "핵꿀" 같은 과격 표현은 자제 |

**비교:**
```
❌ (주식 블로그식)
~로 나타났다. 해당 구간에서 외국인 순매수세가 확인되었다.

✅ (써봄 스타일)
3개월째 쓰고 있는데, 이거 진짜 괜찮다. 특히 맥북 연결이
생각보다 훨씬 안정적이어서 놀랐다.
```

**리뷰의 신뢰도를 높이는 표현들:**
- `처음엔 반신반의했는데...`
- `3개월 써보니 느낀 점은`
- `단점부터 말하자면`
- `A와 B 중 고민한다면, 나는 B를 고르겠다`
- `사진으로만 봤을 땐 몰랐는데 직접 보니`

## 4. 콘텐츠 카테고리 & 비율

## 페이즈별 콘텐츠 전략

### Phase 1: 유입 확보 (1~3개월)

정보성/가이드글로 네이버 검색 유입부터 잡는다. 리뷰는 자연스럽게 정보글 내에 녹이거나 Phase 2부터 본격화.

| 카테고리 | 설명 | 비율 |
|----------|------|------|
| **가이드** | "이렇게 하면 된다" 설정/활용법 | 45% |
| **정보** | IT 트렌드, AI 도구 소개, 생산성 팁 | 35% |
| **리뷰** | 실사용 후기 (정보글 내 자연스럽게 포함) | 10% |
| **비교** | A vs B 비교 | 10% |

### Phase 2: 수익화 본격화 (4~6개월+)

유입이 잡히면 리뷰글 비중을 늘려 쿠팡 전환으로 연결.

| 카테고리 | 비율 |
|----------|------|
| 리뷰 | 25% |
| 비교 | 20% |
| 가이드 | 35% |
| 정보 | 20% |

```

## 왜 이 제품을 골랐나
- 구매 고려하게 된 계기
- 경쟁 제품들과 비교한 이유

## 언박싱 & 첫인상
- 패키징 / 구성품 / 처음 느낌
- 사진 필수
```

## 8. AI 활용 워크플로우 (Enterprise Agent System)

```
Hermes (CEO) — 기획/방향성 결정
  └─ Claude Code — 리서치 + 글쓰기 + 발행
```

**프로세스:**
1. Hermes가 기획 (이번 주엔 어떤 제품?)
2. Claude Code에 리서치 지시 (제품 스펙, 경쟁사, 가격)
3. Claude Code가 리뷰글/가이드글 작성
4. Hermes가 검수 (어조/사실/링크 확인)
5. Claude Code가 썸네일 생성 + 이미지 호스팅
6. Claude Code가 GitHub push / 블로그 발행

**기존 주식 블로그와 차이점:**
- 주식 블로그: Claude Code가 리서치→작성 1:1 담당
- 써봄 블로그: 제품 스펙 리서치(Claude) + 경험 확장(Hermes) → 작성(Claude)
